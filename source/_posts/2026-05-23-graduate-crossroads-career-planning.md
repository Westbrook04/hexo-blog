# 毕业后的十字路口：一次人生战略规划复盘

## 一、背景

2026 届软件工程（智能制造方向）毕业生，目前在一家国网体系的硬件公司做 Java 后端。

听起来还算安稳，但实际处境有不少痛点：

- **每周出差一次**，每次两天，没车，通勤消耗大
- **开发环境差**，无人带，代码规范几乎没有
- **依赖 AI 编码**，独立搭项目能力弱
- **工作时间 955**，请假相对方便——这是为数不多的优势

在这种状态下，必须做一个清晰的战略选择：接下来几年，精力往哪里投？

## 二、摆在面前的选择

当时手上有四条可能的路径：

| 路径 | 吸引力 | 问题 |
|------|--------|------|
| 考研（网安） | 学历提升 | 345分差3分暨大，且当初选网安只因"好考" |
| 换工作（华为OD） | 薪资提升、规范环境 | 机试150/400过线，需集中刷题 |
| 副业（金融自动化） | 长期独立 | 当前精力被工作消耗，无法重投入 |
| 考公/考编 | 稳定 | 时间未到，需换好工作后再启动 |

## 三、最终优先级排序

经过逐一分析，排出了清晰的优先级：

**第一优先级：换工作（华为OD）**
当前出差太消耗精力，必须先把工作换到不出差的环境。华为OD目标院校机试150/400即可通过，门槛不算高，需要的是集中刷题突破。

**第二优先级：副业维持**
和一位北京金融方向的合伙人搭档，我做技术支持，他负责业务。模式是从接项目攒案例开始，不做直接做产品的重投入。保持沟通，等工作稳定后再加码。

**放弃：考研**
考研的目的是为了找更好的工作，但代价是两年时间+机会成本。既然现在就有换工作的机会，考研的优先级就变得很低了。当初选网安也仅是因为"好考"，缺乏真正的学术驱动。

**待启动：考公**
等换好工作、精力恢复后，走非应届赛道考公，计算机专业仍有优势。社保正常缴纳，不纠结应届生身份。

## 四、四周刷题计划

决定了华为OD方向后，制定了四周的集中刷题计划：

| 周次 | 重点 |
|------|------|
| 第一周 | 牛客环境适应 + 数据结构基础 |
| 第二周 | 华为高频中等题突破 |
| 第三周 | 难度提升 + 机试模拟 |
| 第四周 | 面试手撕代码备战 |

几条铁律：
1. **AI 禁用**——刷题期间不依赖 AI，遇到问题先想30分钟再看题解
2. **手写优先**——先在本子上写思路再敲代码
3. **不做 Hard**——时间有限，专注 Medium
4. **每题留记录**——卡点和思路记下来方便复习

## 五、一点复盘

这次决策最有价值的不是"选了哪条路"，而是**看清了什么可以放弃**。

考研是一个典型的例子：它看似是"上进"的选择，但仔细拆解后发现它和最终目标（提高抗风险能力）之间并没有强关联。放弃它不是懒惰，而是把有限的精力集中到真正能改变现状的事情上。

另一个认识是：**不需要一步到位**。先换工作解决眼前的消耗问题，再做下一步规划。每一步都有它的时间窗口。
